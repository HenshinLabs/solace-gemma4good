# MasterLLM v1.0.9

## Highlights

### Model Selection Across Chat And Roleplay
- Added in-session model picker for Roleplay (matching Chat behavior).
- Chat and Roleplay now both list downloaded GGUF and SafeTensors models.
- Selected model is shown directly in the conversation header for faster verification.

### Safer Mid-Conversation Model Switching
- Switching models now resets native runtime safely and reloads with the new model.
- Conversation context is rehydrated into the runtime after model reload so the session can continue with prior history.
- Conversation metadata is updated when model selection changes to keep persistence aligned.

### Runtime Telemetry And Generation Lifecycle Visibility
- Added explicit generation phase status:
  - `Loading model...`
  - `Processing prompt...`
  - `Generating response...`
- Added detailed runtime telemetry cards in Chat and Roleplay:
  - model load duration
  - replayed context message count
  - first-token latency
  - total generation duration
  - decoded output token count and chars
  - decode throughput (tok/s)
  - native runtime speed (tok/s)
  - backend label, thread count, and context size

## Technical Notes
- Chat and Roleplay ViewModels now rehydrate historical system/user/assistant messages after runtime reload.
- SafeTensors handling still validates weights while text generation routes through GGUF fallback where required.
