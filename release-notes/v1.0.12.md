# Master LLM v1.0.12

## Highlights

- Fixed a major GGUF chat freeze caused by token generation executing on the main thread.
- Fixed duplicate native conversation writes that inflated context each turn and degraded performance.
- Hardened native completion lifecycle so cancellation and error paths clean up safely.

## GGUF Runtime Fixes

- `GgufEngine.getResponseAsFlow()` now executes decode work on `Dispatchers.Default` via `flowOn`, preventing UI-thread stalls during token generation.
- `getResponseAsFlow()` and `getResponse()` now always call `stopCompletion()` in `finally`, ensuring native state cleanup even on cancellation/errors.
- Empty token pieces are no longer emitted to the UI stream, reducing unnecessary recompositions.

## Native llama.cpp Bridge Fixes

- `LLMInference` now stores assistant output exactly once per completed generation.
- Added generation completion tracking so partial/cancelled responses are not committed as full assistant turns.
- Removed duplicate assistant insertion at EOG and retained a single commit point in `stopCompletion()`.
- Cleared per-turn response caches at generation start.
- Fixed per-turn batch allocation leak by deleting prior `_batch` before creating a new one.

## Chat / Roleplay Pipeline Fixes

- Removed duplicate `ggufEngine.addUserMessage(...)` calls before `getResponseAsFlow(...)`.
- Removed duplicate `ggufEngine.addAssistantMessage(...)` calls after generation.
- Chat message persistence now happens after model replay/load for text path, preventing duplicate prompt replay on first turn after load.

## SafeTensors Conversion Status

- SafeTensors support in-app remains validator-only for text LLMs.
- On-device conversion from Hugging Face SafeTensors to GGUF or MediaPipe `.task` is still not available in this release.
- Current production path for text inference is:
  - download a GGUF model directly, or
  - convert offline and import GGUF.
