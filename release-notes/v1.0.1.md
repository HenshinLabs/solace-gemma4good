# Master LLM v1.0.1

Release date: 2026-04-12

## Highlights
- Introduced a new branded launcher identity with a production-ready adaptive icon.
- Upgraded release metadata and build outputs for clearer internal distribution.
- Added LeakCanary in debug builds to surface memory leak and heap-retention issues early.
- Fixed Android PackageInstaller crash during APK install (`width and height must be > 0`) caused by icon drawable sizing.

## What Changed

### Branding and APK Visibility
- Replaced placeholder launcher icon layers with adaptive drawable assets for consistent rendering on modern Android launchers.
- Added round icon support in the manifest for devices/launchers that prefer circular masks.
- Updated manifest icon references to a fixed-size vector logo (`@drawable/ic_app_logo`) to ensure install-time icon rendering is stable.
- Replaced launcher background layer with a sized vector drawable to avoid zero-intrinsic-dimension icon layers.
- Added source-controlled SVG logo at `branding/masterllm-logo.svg` to keep design assets under version control.

### Versioning and Artifact Naming
- Bumped app version to:
  - `versionCode = 2`
  - `versionName = 1.0.1`
- Standardized APK filenames generated by Gradle:
  - `MasterLLM-v1.0.1-debug.apk`
  - `MasterLLM-v1.0.1-release.apk`

### Debug Memory Diagnostics
- Added `com.squareup.leakcanary:leakcanary-android:2.14` as a `debugImplementation` dependency.
- Debug builds now auto-include LeakCanary, enabling continuous memory leak observation during development and QA.

## Validation Checklist
- Debug APK assembled with LeakCanary dependency on classpath.
- Release APK assembled with signing config applied.
- Adaptive icon resources compiled and included for launcher visibility.
- Version values and output artifact names verified in build outputs.

## Upgrade Notes
- No migration steps are required for existing users.
- QA should re-install using the new debug/release APK names to avoid confusion with prior artifacts.
