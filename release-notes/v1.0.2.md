# Master LLM v1.0.2

Release date: 2026-04-12

## Highlights

- Fixed Hugging Face token validation by normalizing pasted tokens (including optional `Bearer` prefix) and validating against `whoami-v2` with fallback to `whoami`.
- Fixed model download auth headers for gated repos by reusing normalized token handling.
- Updated model storage behavior to use a Download-based root by default and preserve nested per-repo/per-file folder structure.
- Added support for custom model storage root from Settings while keeping safe fallback behavior.
- Improved Auth screen top layout scaling and status bar handling for cleaner edge-to-edge rendering.
- Added per-prompt runtime stats in Chat:
  - Prompt/output token counts
  - Output chars
  - Latency
  - Tokens per second
  - Backend mode, thread count, GPU layer count, and context size
- Added GPU driver diagnostics in Settings Runtime section:
  - Adreno detection
  - Turnip asset readiness
  - Native backend availability
  - Active runtime mode and ICD path details
- Wired thread count and GPU acceleration settings into GGUF runtime configuration for both Chat and Roleplay generation flows.

## Validation scope

- Unit tests added for:
  - Hugging Face token normalization helper
  - Auth login token normalization and whoami-v2 fallback behavior
  - Marketplace custom storage path behavior
- Existing marketplace and runtime tests retained.
