# MasterLLM v1.0.6 Pre-Release

## What's New

### LLM Model Configuration UI

**Chat & Roleplay Model Settings Dialog**
- Access via settings icon in chat toolbar
- Configure all inference parameters:
  - **Temperature** (0.0 - 2.0): Controls randomness of output
  - **Top P** (0.0 - 1.0): Nucleus sampling threshold
  - **Top K** (1 - 100): Top-K sampling
  - **Repeat Penalty** (1.0 - 2.0): Penalty for repeating tokens
  - **Max Tokens** (64 - 8192): Maximum generation length
  - **System Prompt**: Custom system instructions
- Reset to defaults button
- Parameters are saved per conversation

### Inference Engine Improvements

**KV Cache Management**
- Added `clearKVCache()` method for memory management
- Proper cache clearing between conversations

**Tokenization Support**
- Added `tokenize()` method for native tokenization
- Added `getVocabSize()` to query model vocabulary size
- Improved token counting for accurate stats

**Native JNI Bridge**
- Full JNI implementation with native methods:
  - `nativeLoadModel()` - Load GGUF models
  - `nativeUnloadModel()` - Unload and cleanup
  - `nativeGenerate()` - Synchronous generation
  - `nativeGenerateTokens()` - Streaming generation
  - `nativeTokenize()` - Tokenize text
  - `nativeClearKVCache()` - Clear KV cache
  - `nativeGetVocabSize()` - Get vocab size
  - `nativeSetEnv()` - Set environment variables

### Chat System Enhancements

**ChatViewModel Updates**
- Added `InferenceParams` to state
- Actions for updating each parameter
- Model selection properly triggers reload
- Generation passes params to engine
- Model configuration dialog UI

**RoleplayViewModel Updates**
- Same inference parameter support as chat
- Parameters integrated with roleplay sessions

## Technical Details

### Architecture

**Kotlin Layer**
- `GgufEngine` - Main interface for model operations
- `ChatViewModel` - Manages chat state and generation
- `RoleplayViewModel` - Manages roleplay state

**Native Layer** (JNI)
- `gguf_bridge_stub.cpp` - JNI bridge implementation
- Context management with thread-safe maps
- Token streaming via callbacks

**State Management**
- `InferenceParams` data class for parameters
- `ChatUiState` includes configuration dialog state
- `RoleplayUiState` includes parameters

## Known Issues

### Native Implementation
The current release uses a **stub native implementation** that provides:
- Context management
- Token streaming simulation
- Response generation based on prompt keywords

**For production inference**, replace `gguf_bridge_stub.cpp` with the full llama.cpp integration in `gguf_bridge_full.cpp`, which requires:
- llama.cpp built from source via FetchContent
- Proper NDK configuration
- Device with sufficient RAM (4GB+ recommended)
